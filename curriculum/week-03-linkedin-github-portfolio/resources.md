## Tools you'll use this week

- **Your existing LinkedIn account.** No premium subscription needed for any of this.
- **Your existing GitHub account.** All edits are to repos you already own, plus a new `<username>/<username>` profile-README repo.
- **A peer who can search your name** and report what they see — for the recruiter-skim test.
- **A text editor for Markdown** — VS Code, Cursor, GitHub's web editor, or anything else. The READMEs are short.
- **A clean private-browsing window** for testing what an outside viewer sees on your LinkedIn (logged-out view differs from logged-in).

## Glossary

| Term | One-line definition |
|------|---------------------|
| **Headline** | The single line under your name on LinkedIn. 220 chars. The most-read field on your profile. |
| **About** | LinkedIn's free-text summary. 200-300 words, written as narrative, not bullets. |
| **Open to Work** | A LinkedIn signal (banner ring or recruiter-only) marking you as actively searching. |
| **Profile README** | A `<username>/<username>` GitHub repo whose README displays on your profile homepage. |
| **Pinned repo** | One of up to six repos that appear above the fold on your GitHub profile. |
| **Project README** | The README inside a specific repo. The first thing a reviewer reads when they click in. |
| **Fork** | A copy of someone else's repo on your account. Counts toward your repo count; signals nothing on its own. |
| **Activity graph** | The green-square contribution grid on your GitHub profile. Visible to all viewers. |
| **Cross-surface consistency** | The property that your resume, LinkedIn, and GitHub tell the same story without contradiction. |
| **The recruiter skim** | The 60-90 second click-through a recruiter does: name search, LinkedIn glance, GitHub glance. |
