## Problem 1 — Cross-surface consistency audit (45 min)

Open three surfaces side by side: your v2 resume PDF, your updated LinkedIn profile, and your GitHub profile. In `c13-week-03/cross-surface-audit.md`, produce a table that compares them.

| Field | Resume says | LinkedIn says | GitHub says (profile README / bio) | Consistent? |
|-------|------------|---------------|-----------------------------------|-------------|
| Current title |  |  |  |  |
| Current company |  |  |  |  |
| Years of experience implied |  |  |  |  |
| Top 3 technologies |  |  |  |  |
| Primary domain (payments / ML / etc.) |  |  |  |  |
| Most prominent project / outcome |  |  |  |  |
| Location |  |  |  |  |
| Contact email |  |  |  |  |

**Acceptance:**

- Every row filled in.
- Each "Consistent?" cell is `Yes`, `Minor` (small wording difference, same meaning), or `No` (substantive disagreement).
- For every `No`, a one-line decision: which surface gets updated to match, and why.
- A 50-word summary at the top: how consistent was the picture, and which surface was most often out of date?

## Problem 2 — Polish the second pinned repo (60 min)

Exercise 3 polished the **main project** repo. This problem polishes the **breadth** repo from Exercise 2 — the second of your three pinned slots.

The breadth repo's README does not need to be as long as the main project's. Target: 100-300 lines of markdown. Apply the same 8-section structure from Lecture 2, §6, but shorter.

**Acceptance:**

- Breadth repo's README is rewritten using the 8-section structure.
- Quick start (3 commands or fewer) tested on a fresh clone.
- A screenshot, GIF, or sample output exists and is referenced.
- LICENSE present.
- Commit message in the repo: `Polish README for portfolio (C13 Week 3)`.

## Problem 3 — Recommendations + Skills cleanup on LinkedIn (45 min)

Two targeted LinkedIn fixes that pay off across the cycle.

**Part A — Skills cleanup:**

- Open your LinkedIn Skills section.
- Re-order the top 3 to be the three skills you most want recruiters to filter on.
- Remove every skill you wouldn't comfortably discuss in a 30-minute screen.
- Add any technology that appears in your resume but not in Skills.

**Part B — Request 1-2 recommendations:**

- Identify 2-3 past colleagues (manager, peer, or report) who would write you an honest recommendation.
- Send each a request. Suggest a topic ("could you mention the X project, since that's where we worked closely?"). Personal request beats LinkedIn's default template.

**Acceptance:**

- `c13-week-03/linkedin-skills-and-recs.md` containing:
  - Before-and-after Skills list (top 10).
  - Who you requested recommendations from + the message you sent (anonymized initials if needed).
  - Expected timeline (most people take 1-2 weeks; that's fine).

## Problem 4 — Read three engineer profiles at your target level (45 min)

Find three public LinkedIn or GitHub profiles of engineers at the level you're targeting, ideally at companies on your Week-1 target list. Use LinkedIn search, your Week-1 contacts, or GitHub's "popular contributors" pages.

For each profile, write:

- Link / source.
- What's strong about their public surface (headline, About, pins, READMEs).
- What's weak or what you'd do differently.
- One pattern you're going to steal for your own surfaces.

**Acceptance:**

- `c13-week-03/competitor-profiles.md` with three sections — one per profile — covering all four points above.
- At least one of the "stolen patterns" applied to your own profile before submitting the mini-project.

## Problem 5 — The "Google your name" audit (40 min)

In a private browser window, search:

- `"Your Full Name"`
- `"Your Full Name" engineer`
- `"Your Full Name" {your-city}`
- `"Your Full Name" {your-current-employer}`

For each search, note the first 5 results.

**Acceptance:**

- `c13-week-03/google-audit.md` listing:
  - The four searches and their top 5 results each.
  - For each result that isn't your LinkedIn / GitHub / personal site: what is it, and is it a problem?
  - A list of "addressable" items (old accounts to lock down, namesake confusion to disambiguate, stale public pages to update).
  - For each addressable item, the action: delete / make private / leave / push more recent signal to outrank.
- At least one action taken in the week.

## Problem 6 — Week-3 reflection (40 min)

`c13-week-03/reflection.md`, 300-400 words:

1. Before this week, what did your public surface (LinkedIn + GitHub) tell a recruiter about you that was *wrong* or *missing*? Be specific.
2. Of the three exercises this week, which one moved the needle most for you, and why?
3. If a recruiter Googled your name today, would the first 3 results convince them to schedule a screen? Defend your answer with evidence from the peer skim test (Challenge 1).
4. What one habit are you going to carry into Week 4 — concrete, testable, not aspirational? (Example: "Every time I update my resume's master, I'll spend 5 minutes propagating the change to LinkedIn the same day.")
