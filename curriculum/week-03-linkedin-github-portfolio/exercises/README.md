# Week 3 — Exercises

Three exercises. Each operates on the public surface a recruiter or hiring manager will actually see.

1. **[Exercise 1 — Rewrite headline and About](exercise-01-rewrite-headline-and-about.md)** — produce a new LinkedIn headline (3 candidates, one chosen) and an About section that maps to your resume (~60 min)
2. **[Exercise 2 — Pin three best repos](exercise-02-pin-three-best-repos.md)** — audit your GitHub, choose three pinned repos, set up your profile README (~75 min)
3. **[Exercise 3 — README for your top project](exercise-03-readme-for-your-top-project.md)** — write a real README for the strongest of your three pinned repos (~75 min)

## How to work them

Do them in order. Exercise 1 is text-only, can be done from anywhere, and shapes the LinkedIn surface that drives recruiter inbound. Exercise 2 is the audit and curation step on GitHub — it surfaces *which* repo deserves the deep README work in Exercise 3. Exercise 3 is the load-bearing piece: one repo, one excellent README, the artifact a hiring manager will actually read.

**Don't claim a polished public surface until all three exercises and the mini-project peer review are done.** A great headline with an empty GitHub still loses to a mediocre headline with a thoughtful pinned project. The combination is what wins.
