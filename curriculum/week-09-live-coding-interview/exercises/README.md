# Week 9 — Exercises

Three exercises. Each isolates one under-rehearsed phase of the live coding round and drills it on its own, with a recording and a self-scored rubric.

1. **[Exercise 1 — The clarification drill](exercise-01-the-clarification-drill.md)** — three 8-minute rounds on the clarify-and-examples phase only, no code written, scored on a four-axis rubric (~30 min)
2. **[Exercise 2 — The stuck-recovery drill](exercise-02-the-stuck-recovery-drill.md)** — one closed-book 40-minute round on a problem one notch above your comfort, rehearsing the three stuck-recovery moves (verbalise, clarify, simplify) when you hit the wall (~45 min)
3. **[Exercise 3 — The complexity-defence drill](exercise-03-the-complexity-defence-drill.md)** — six 4-minute rounds analysing unfamiliar code snippets for time, space, optimality, and one trade-off each (~30 min)

[`SOLUTIONS.md`](SOLUTIONS.md) holds the worked solutions and answer keys for the drill problems. Use it only after the timer ends on each round — the drill is the running, not the reading.

## How to work them

The three exercises are independent reps on three different phases, so the order is flexible, but the natural progression follows the round itself: clarify (Exercise 1) → solve under pressure (Exercise 2) → defend the complexity (Exercise 3). Each is short and recorded, and the recording is the artefact you grade against — watch it back, count the filler words, and note the one missed clarifying question or the one stuck-point pattern that recurs.

Exercise 2 is the one most candidates have no rehearsed response to, because solo LeetCode lets you hit the hint button instead of staying stuck. Run it closed-book: no hints, no editorial, no LLM. If your recording contains zero stuck moments, the problem was too easy — pick a harder one; the drill only works at the edge of your competence.

These are repeat drills, not one-and-done. Run Exercise 1 weekly for the first three weeks of an active cycle (then before any high-stakes round); run Exercise 2 the morning of an onsite, 6-8 hours before the first round, as a warm-up; run Exercise 3 monthly, generating fresh snippets from any LeetCode Medium. The exercises also set up the week's challenges, which drill the two failure modes — ignoring the hint, over-engineering — that need a human or a side-by-side comparison rather than a solo timer.
